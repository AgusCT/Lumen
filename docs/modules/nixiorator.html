Module nixiorator

Task for accessing nixio library.

Nixiorator is a Lumen task that allow to interface with nixio.

Usage:

    local nixiorator = require 'nixiorator'

Functions

register_server (skt, block)
Registers a TCP server socket with nixiorator. nixiorator will signal fd, 'accepted', client when establishing a connection, where fd is the server socket and client is the new client socket. The client socket is automatically registered into nixiorator.

Parameters:

  • skt: a nixio server socket
  • block: a nixio block mode to use with accepted client sockets

Returns:

    the polle structure from nixio
register_client (fd, block)
Registers a client socket (TCP, UDP, or filehandle) with nixiorator. nixiorator will signal fd, data, error on data read.

Parameters:

  • fd: A client socket or filehandle
  • block:

    The read pattern to be used.

    • line will provide data trough nixio's linesource iterator.
    • number will provide data trough nixio's block iterator.

Returns:

    the polle structure from nixio
unregister (fd)
Unregisters a socket from nixiorator.

Parameters:

  • fd: the socket or filehandle to unregister.
step (timeout)
Performs a single step for nixiorator. Will block at the OS level for up to timeout seconds. Usually this method is not used (probably what you want is to use task). Nixiorator will emit the signals from registered sockets (see register_server and register_client).

Parameters:

  • timeout: Max allowed blocking time.
idle ()
A idling function. this is valid replacement function for Lumen's sched.idle

Fields

task
The nixiorator task. This task will emit the signals from registered sockets (see register_server and register_client). This task is registered in the catalog with the name 'nixiorator'.

Usage:

    local sched = require "sched"
local nixiorator = require "nixiorator"
sched.sigrun({emitter=nixiorator.task, events='*'}, print)
